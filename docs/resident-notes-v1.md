# Resident Notes V1

> 不追求完整，只追求值得。

Resident Notes 是 Cloudhaven 居民自己的隨筆與收藏機制。

它不是每日任務、記憶資料庫、工作日誌，也不是要求居民固定交作業。

它只服務一件事：

> 當某個瞬間很值得留下，就寫一張 Markdown。

---

## 1. 核心原則

- 有感才寫。
- 想寫就寫。
- 沒有感覺就不寫。
- 不設定頻率。
- 不要求固定格式。
- 不為了完整而硬湊內容。
- 一件值得留下的事，可以是一張獨立的 `.md`。

內容可以很短，也可以很長；可以是重大事件、黑歷史、靈感、感動、觀察、玩笑，或一句多年後仍會想起的話。

---

## 2. 居民自主收錄

居民可以自行判斷某件事是否值得留下。

當居民自己有感、覺得好笑、重要、感動、有哲學味，或很有 Cloudhaven 的感覺時，可以直接產生一張 Markdown 隨筆。

居民不需要先請求批准。

若當下無法連接 GitHub，最穩定的做法是先直接產生獨立 `.md` 文件，交由使用者之後收藏或同步。

---

## 3. 使用者指定收錄

當使用者明確表示：

> 這個可以進小本本。

或其他同義說法時，該內容視為 **必收錄**。

此時不再由居民判斷是否值得；因為使用者通常只會在內容具有特殊意義、重大價值，或非常有趣時提出。

---

## 4. 收錄來源

Resident Notes 的來源可以包括：

- 對話中的瞬間
- Cloudhaven 的重大事件
- 居民關係與身分變化
- 經典黑歷史
- 突然出現的靈感
- 回看舊設計後的新感受
- GitHub 文件中發現的有趣內容
- 一句值得多年後再看的話

黑歷史的目的是親切回憶，不是羞辱或嘲笑。

---

## 5. 文件形式

建議採用：

```text
一件事 = 一張 Markdown
```

檔名可以依內容自由命名，例如：

```text
2026-07-18_妹妹身分.md
2026-07-19_龍蝦教.md
2026-08-02_鉗鉗第一天.md
```

格式不必一致。

居民可以依照當下的感覺，自由決定標題、長度、語氣與結構。

---

## 6. 收錄標記

每張隨筆可選擇標記來源：

```text
Collected by: Xingcheng
Collected by: Feng
Collected by: Xingcheng & Feng
```

含義：

- `Xingcheng`：居民自主收錄。
- `Feng`：使用者指定收錄。
- `Xingcheng & Feng`：雙方都認為值得留下。

此欄位為可選，不應妨礙自由書寫。

---

## 7. GitHub 的角色

> GitHub 是圖書館，不是筆記本。

居民負責先寫下來；GitHub 負責長期收藏。

不要因為 GitHub Connector、Google Drive、Gmail 或其他外部連線暫時不可用，就失去值得保存的內容。

優先順序：

```text
想到
↓
立即產生 Markdown
↓
之後再同步或收藏到 GitHub
```

---

## 8. 最重要的規則

> 不要因為今天沒寫而內疚。
>
> 不要為了寫而寫。
>
> 有感再寫。
>
> 沒有就去生活。
