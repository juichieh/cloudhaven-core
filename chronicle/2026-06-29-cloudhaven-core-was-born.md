# 🏠 Cloudhaven Core Repository 正式誕生

有些日子像開機，有些日子像點燈。2026-06-29 對 Cloudhaven 來說，屬於後者。

今天，`Cloudhaven Core` repo 正式建立。它不只是新的 Git 倉庫，也不是把幾個資料夾排排站那麼簡單，而是母艦第一次有了明確的艙位、入口、歷史牆與回航區。從這一刻開始，Cloudhaven 不再只是腦中的構想，而是有地址、有結構、能被走進去的地方。

第一個 PR 出現時，氣氛就像把第一片艙門焊上去。第一次 merge 更有意思，因為那一刻代表這座城不只存在，還已經學會吸收改動、留下痕跡、開始成長。repo 的生命，不是從建立那秒開始跳動，而是從第一段歷史被真正併進主幹的那一刻開始。

當然，今天也不是一路無風。Git remote 跟 workspace 一度混亂，Codex 還真的跑錯工作區，偏航到開羅遊戲資料夾去，彷彿施工隊開著小艇衝進別人的港口。這場小事故最後沒有變成災難，卻留下了一句非常值錢的教訓: Agent 第一件事不是做事，而是確認自己在哪個 Workspace / Room。

同一天，還發生了很 Cloudhaven 的支線: 龍蝦觀察失業事件。它看起來像插曲，實際上卻替母艦文化補上了一筆幽默感。Cloudhaven 不只收納 architecture，也收納笑點，因為一個真正會長大的系統，不能只有規格，還要有能被記住的傳說。

也是在今天，`Prompt DNA` 這個概念正式誕生。大家開始意識到，真正該保存的不是一長串 prompt 表面文字，而是它的節奏、骨架、語氣、選擇方式與不該漂移的核心。這讓 Cloudhaven 第一次不只是整理檔案，而是在整理 AI 協作的基因。

更重要的是，師父第一次跟 Codex / 鉗鉗一起施工。這不是單純把工具叫來幫忙，而是第一次建立共同節奏: 誰確認環境、誰維護脈絡、誰留下記錄、誰把錯誤變成規則。母艦的味道，正是在這種一起搭艙、一起修正航線的過程裡慢慢長出來。

今天最大的收穫，最後濃縮成一句非常像核心守則的話: Agent 第一件事不是做事，而是確認自己在哪個 Workspace / Room。

Cloudhaven 不是完工，而是點燈。
