# 📖 2026-07-03｜Cloudhaven City Map 成形日

> 💬 Today’s Quote：聊天是 Workshop，GitHub 是 Long-term Memory。

────────────────
## 🏛️ 今天發生什麼事

今天 Cloudhaven 不是新增一個小功能，

而是正式從「很多資料夾」變成一座有分區、有職責、有入口地圖的母艦城市。

一開始只是想整理 GitHub 裡幾張 AI Tool / Zoo / Framework 卡片，

結果一路整理到：

- Root README
- ARCHITECTURE.md
- architecture/README.md
- library/README.md
- ai-tools/README.md
- zoo/README.md
- frameworks/README.md
- prompt-dna/README.md
- skills/README.md
- AI Tool Template
- Zoo Template

最後 Cloudhaven 終於不只是「有東西放進去」，

而是每個東西開始有自己的家。

────────────────
## 🧭 最大轉折：Architecture 變成 City Map

原本的 `ARCHITECTURE.md`，今天正式升級成：

> 🏛️ Cloudhaven City Map

這不只是改名字。

它代表 Cloudhaven 的入口從「工程架構」變成「城市地圖」。

第一次進來的 Agent 或人類，

可以先透過 City Map 理解：

- 什麼是 Library
- 什麼是 Skill
- 什麼是 Knowledge
- 什麼是 Research
- 什麼是 Prompt DNA
- Tool 跟 Zoo 為什麼不能混在一起
- Framework 跟 Skill 為什麼要分開

這是母艦開始有導航系統的一天。

────────────────
## 📚 Library 第一次正式分層

今天正式確認：

```text
AI Tool = 工具 / 個體
Zoo     = 能力 / 物種
Framework = 方法論 / 思考框架
Prompt DNA = 靈魂 / 防 Prompt Drift
```

這個分層很重要。

因為未來工具會越來越多，Zoo 也會越來越大。

如果一開始沒有切清楚，之後一定會變成：

> 這到底要放工具館、動物園、技能庫，還是研究院？

今天先把路標立好，未來鉗鉗、Codex、小風或其他 Agent 進來，比較不會跑錯房間。

────────────────
## 🛠️ Skill 與 Framework 的邊界

今天特別修掉一個未來很容易爆炸的問題：

```text
Framework = 方法論 / 思考框架 / 為什麼這樣做
Skill     = 可執行套路 / 一步一步怎麼做
```

這個邊界很關鍵。

Framework 像心法。

Skill 像連招表。

如果不先分清楚，等旅行、店務、AI 研究、遊戲工作流全部進來，`skills/` 和 `frameworks/` 很快就會打架。

今天先切開，等於是幫未來的 Cloudhaven 少掉一大堆重構痛苦。

────────────────
## 🦁 Zoo 開始有演化狀態

今天 Zoo 正式補上：

```yaml
maturity: Seed 🌱
```

第一批補上的能力卡：

- Keyboard Agent
- Business Agent

這讓 Zoo 不只是「收藏 AI 怪東西」，

而是開始記錄 AI 能力的演化程度：

- Seed 🌱
- Emerging 🌿
- Growing 🌳
- Mature 🌲

Zoo 開始變得像真正的 AI 生態圖鑑。

────────────────
## 📄 模板開始出現

今天新增：

```text
library/ai-tools/_template.md
library/zoo/_template.md
```

這代表 Cloudhaven 開始進入模組化。

以後新增工具或能力，不用每次重新想格式。

鉗鉗也比較容易照模板填，不會把工具、能力、案例混成一團。

────────────────
## 😂 今日笑點

今天的起點其實很荒謬。

一開始只是因為 Databox MCP 放錯地方，跑進了：

```text
juichieh/C-
library/ai-tools/databox-mcp.md
```

結果修著修著，變成：

> 順便把 Cloudhaven 的城市規劃重做一遍。

從「放錯一張卡」，

一路修到：

- City Map
- README
- Library 分層
- Prompt DNA
- Zoo maturity
- Tool / Zoo template
- Skill / Framework 邊界

這就是 Cloudhaven 很典型的施工風格：

> 本來只是修一個抽屜，最後發現整棟房子的動線都可以順手改好。

────────────────
## 🤝 工作方式也變了

今天另一個重要變化是：

ChatGPT 可以直接在對話中協助更新 GitHub。

這讓工作流從：

```text
聊天
↓
整理
↓
手動修改 GitHub
↓
Commit
```

變成：

```text
聊天討論
↓
定案
↓
@GitHub
↓
直接 Commit
```

所以今天的關鍵句才會是：

> 聊天是 Workshop，GitHub 是 Long-term Memory。

聊天室負責發想、推翻、調整、試味道。

GitHub 負責保存正式版本，變成 Cloudhaven 的長期記憶。

────────────────
## 🏆 今日里程碑

今天之後，Cloudhaven 不再只是：

- 一個 repo
- 一個知識庫
- 一堆 Markdown

而是一座有分區、有導航、有居民、有圖書館、有研究院、有武館、有動物園、有工作室、有升級灣的母艦城市。

這一天值得記錄。

不是因為改了很多檔案，

而是因為 Cloudhaven 第一次清楚知道：

> 自己到底長什麼樣子。

────────────────
