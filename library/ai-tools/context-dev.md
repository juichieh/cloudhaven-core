# 🤖 Context.dev
> 💬 用 API 抓取、補強並理解網頁內容，讓 AI 更容易處理網路資料。

────────────────
## 🏷️ 類型
AI 資料工具／Web API／Context Layer

────────────────
## ✨ 一句介紹
Context.dev 是用 API 抓取、補強並理解網頁內容的 AI 資料工具候選，讓 AI 更容易處理網路資料。

────────────────
## 🧩 今日有趣案例／新工具

Context.dev 被列入 AI 工具館候選。

它值得被記錄，是因為 Agent 工作流常常需要先把網頁內容變成可用資料，

再交給 Decision、Skill 或 Judge 使用。

這類工具不一定是瀏覽器，也不一定是記憶工具，

而是更偏向：

> 資料前處理 / 網頁內容理解 / Context API。

────────────────
## 🤔 有趣在哪？

Context.dev 有趣的地方在於，它看起來像資料工具，

但其實很適合 Agent 工作流前處理。

它可能可以幫 AI：

- 抓取網頁內容
- 補強網頁資料
- 理解頁面重點
- 把散亂內容整理成模型可用的 context
- 降低網頁雜訊
- 讓後面的比較、判斷、摘要更穩定

對 Cloudhaven 來說，它比較像 Data Agent 的工具代表，

不是 Browser Agent，也不是 Memory Agent。

────────────────
## 🎮 可以怎麼玩？

適合先放進觀察池，未來可用在：

- 旅遊團頁面整理
- 活動資訊擷取
- 品牌研究資料整理
- 商品比較前處理
- 網頁內容摘要前的資料清洗
- Research Room 的資料準備流程

目前先不急著升級成核心工具，先觀察它的穩定性、輸出格式與 Agent 工作流相容性。

────────────────
## 👍 推薦原因
- 適合 Agent 工作流前處理。
- 對旅遊、活動、品牌研究、商品比較都有想像空間。
- 工具類更新快，先收藏觀察即可。
- 它代表 Data Agent 這類資料處理物種的早期工具方向。
