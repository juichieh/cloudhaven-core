# 🤖 Contextberg / scritty 類記憶工具
> 💬 幫 Coding Agent 或工作 Agent 保留可搜尋專案脈絡的工具方向。

────────────────
## 🏷️ 類型
🧠 Agent Memory

Agent 記憶工具／Coding Agent 上下文工具／專案脈絡索引層。

────────────────
## ✨ 一句介紹
Contextberg / scritty 類記憶工具，是讓 Coding Agent 或工作 Agent 保留可搜尋專案脈絡的工具候選。

────────────────
## 🧩 今日有趣案例／新工具

Contextberg / scritty 類記憶工具被列入今日新增圖鑑候選。

它值得注意，是因為 Agent 開始不只需要「一次任務能力」，

也需要能在下次工作時接上之前的脈絡。

這非常接近澄丸的核心方向：

> 不是記很多，而是下次做事能接得上。

────────────────
## 🤔 有趣在哪？

這類工具有趣的地方在於，它把記憶從聊天紀錄變成工作交接基礎設施。

對 Coding Agent 或長期工作 Agent 來說，真正重要的不是保存所有內容，

而是保存任務接續需要的線索：

- 之前做到哪裡
- 哪些檔案動過
- 哪些規則已經定案
- 哪些問題已經處理過
- 哪些下一步要接續
- 哪些決策已經完成

這會讓 Agent 從「每次重新開始」變成「可以長期接力」。

────────────────
## 🎮 可以怎麼玩？

Cloudhaven 可以先觀察這類工具是否適合：

- Coding Agent 專案交接
- 鉗鉗任務接續
- 旅遊比較專案脈絡
- Cloudhaven 架構決策保存
- Session Handoff
- GitHub / Obsidian / local workspace 的脈絡橋接

測試重點不是看它記了多少，

而是看它下次能不能真的幫 Agent 接上任務。

────────────────
## 👍 推薦原因
- 它非常接近澄丸的核心定位。
- 它能改善 Agent 跨 session 斷片問題。
- 它適合長期專案、Coding Agent、研究 Agent 與工作 Agent。
- 它提醒 Cloudhaven 記憶層要重視搜尋、摘要、淘汰與接續，而不是只堆資料。
