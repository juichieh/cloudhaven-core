# 🤖 BrowserAct
> 💬 給 AI agent 理解與處理公開網頁情境的瀏覽器層候選工具。

────────────────
## 🏷️ 類型
🌐 Browser Agent Infrastructure

Browser Agent 工具／瀏覽器研究層／Web Agent 基礎設施。

────────────────
## ✨ 一句介紹
BrowserAct 是給 AI agent 處理真實網頁情境的瀏覽器層工具候選，讓 Agent 有機會從聊天介面走向公開網頁世界。

────────────────
## 🧩 今日有趣案例／新工具

BrowserAct 被列入今日新增圖鑑候選。

它值得被注意，不是因為它只是另一個瀏覽器工具，

而是因為它代表 Agent 開始需要一個能安全、穩定、可觀察地理解公開網頁的中介層。

對鉗鉗來說，這類工具可能會成為未來的「網路手腳」參考。

────────────────
## 🤔 有趣在哪？

BrowserAct 有趣的地方在於：

Agent 不再只是讀使用者貼進來的內容，

而是可以開始面對真實網站頁面、整理公開資訊、建立研究脈絡。

這讓許多 Cloudhaven 任務變得更實際：

- 查旅行社行程
- 比較飯店資訊
- 觀察 Steam / GitHub / 文件頁
- 擷取公開頁面重點
- 建立可回顧的網頁研究紀錄

但這也代表安全與權限會變得很重要。

瀏覽器能力越強，越需要白名單、回報、觀察與人類確認。

────────────────
## 🎮 可以怎麼玩？

可以先用低風險研究任務觀察：

- 只讀公開網頁
- 只在白名單網站測試
- 讓 Agent 整理公開資訊
- 記錄每一步研究脈絡
- 和 Judge / Permission 搭配使用

適合 Cloudhaven 先作為 Browser Agent Infrastructure 的候選工具收藏。

────────────────
## 👍 推薦原因
- 它代表 Agent 從聊天介面走向公開網頁世界。
- 它可能成為鉗鉗未來「網路手腳」的重要參考。
- 它能支援旅遊、遊戲、研究、GitHub 文件等公開資料任務。
- 它也提醒 Cloudhaven 必須提早設計瀏覽器白名單與安全邊界。
