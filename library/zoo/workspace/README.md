# 🏢 Workspace／工作空間

> 觀察 Agent、工作流、MCP、知識庫、人工審批與企業服務如何被整合成同一個工作環境。

────────────────

## 🏷️ 分類定位

Workspace / Open-source Agent Platform / Agent Operating Environment

這類卡片主要觀察：

- Agent 是否從單體工具變成可協作的工作台
- Agent、MCP、知識庫與自動化流程是否能共同運作
- 狀態、權限、審批、重試與錯誤處理如何被整合
- 是否開始出現真正的「Agent 居住環境」

────────────────

## 🧭 適合收錄

- Workspace
- Multi-Agent Workbench
- Open-source Agent Platform
- Agent Workflow Studio
- Agent Operating Environment
- Human-in-the-loop Workspace

────────────────

## 🧠 命名說明

這裡使用 `Workspace`，不使用 `Agent Workspace`。

Cloudhaven Zoo 的命名原則：

- 代表一種 Agent 類型時，使用 `XXXX Agent`
- 代表能力、基礎設施、環境或概念時，直接使用專有名詞
- 不為了格式一致而硬加 `Agent`
