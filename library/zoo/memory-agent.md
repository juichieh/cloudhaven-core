# 🦁 Memory Agent
> 💬 讓 Agent 跨 session 還記得自己在做什麼的長期接力物種。

────────────────
## 🏷️ 能力類型
🧠 Agent Memory

Agent 記憶層／跨 Session 工作記憶／可搜尋長期上下文。

────────────────
## ✨ 一句介紹
Memory Agent 是能讓 Coding Agent 或工作 Agent 跨 session 保留可搜尋記憶、讓下次做事能接得上的 Agent 物種。

────────────────
## 🧠 這是什麼能力？

Memory Agent 不是單純把所有聊天記下來。

它更接近一種「任務接力型 Agent」。

重點不是記很多，

而是讓 Agent 下次能知道：

- 之前做到哪裡
- 哪些決策已經定案
- 哪些坑踩過
- 哪些檔案改過
- 哪些規則不能忘
- 下一步應該從哪裡接續

對 Cloudhaven 來說，這非常接近澄丸的核心能力。

不是收集一堆雜訊，

而是保留能讓下次工作銜接上的關鍵記憶。

────────────────
## 🤔 為什麼重要？

Agent 越常工作，越需要記憶。

沒有記憶的 Agent 每次都像新來的實習生，

有記憶的 Agent 才能真正長出：

- 長期專案感
- 個人工作習慣
- 已知限制
- 已完成決策
- 可搜尋任務上下文
- 跨 session 的連續性

這對 Coding Agent、研究 Agent、旅行比較 Agent、店務 Agent 都很重要。

尤其是 Cloudhaven 這種長期演化的母艦，記憶不是裝飾，而是基礎設施。

────────────────
## 🧬 演化狀態
🌱 Seed / 萌芽

目前屬於可研究階段。

原因：

- 記憶品質比記憶數量更重要
- 需要處理噪音、過期資訊、權限與隱私
- 要能被搜尋、摘要、更新與淘汰
- 真正有價值的是「下次接得上」

可選狀態：

- Seed 🌱：剛出現，只有少數早期案例
- Emerging 🌿：開始有多個案例，方向變清楚
- Growing 🌳：生態快速長大，開始影響主流使用方式
- Mature 🌲：已經成熟，成為穩定基礎能力

────────────────
## 📈 觀察重點
- 是否能跨 session 穩定保留任務上下文
- 是否支援搜尋與摘要，而不是只堆資料
- 是否能區分短期工作記憶與長期記憶
- 是否能淘汰過期資訊，避免記憶污染
- 是否適合澄丸 / Cloudhaven 的記憶層設計
- 是否能和 GitHub / Obsidian / local workspace 串接
