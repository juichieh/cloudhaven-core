# 🧬 Agent Memory／記憶型居民

> 記憶讓 Agent 成為長期居民，也讓跨時間污染、來源不明與延遲攻擊成為新的治理問題。

## 🏷️ 分類

🧠 數位居民記憶／可追溯長期記憶

## 🌿 成熟度

🌿 從跨系統便利層，演化成需要來源治理的安全基礎設施

## ⭐ 收藏價值

★★★★★

## 🧬 和上次相比改變了什麼？

前期主要觀察：

- 跨 App
- 跨模型
- 跟著居民搬家
- 形成持續身分

本次新增的關鍵演化是：

> 記憶不只是身分與便利性的基礎，也正式成為跨時間攻擊面。

GhostWriter 顯示，惡意內容可以先潛伏在記憶中，再由未來任務喚醒；Agent Data Injection 則提醒，外部假資料可能繞過只針對惡意「指令」設計的防線。

原本這個物種是：

> 記得越多，越像長期同伴。

今天演化成：

> 記得越久，越需要知道每段記憶從哪裡來。

## 🎯 新增治理要求

- 每段記憶的來源與時間戳
- 信任等級與寫入權限
- 使用前重新驗證
- 隔離、刪除、回滾與稽核
- 外部資料與內部記憶的邊界
- 跨 App 同步時的污染傳播控制
