# 🦁 Browser Agent Infrastructure
> 💬 讓 AI Agent 從聊天框走進真實網頁世界的瀏覽器手腳。

────────────────
## 🏷️ 能力類型
🌐 Browser Agent Infrastructure

瀏覽器代理基礎設施／Web 操作層／Agent 網路手腳。

────────────────
## ✨ 一句介紹
讓 AI agent 可以操作真實網頁、讀取頁面、點擊、填表、查資料與完成跨網站任務的瀏覽器層能力。

────────────────
## 🧠 這是什麼能力？

Browser Agent Infrastructure 不是單一工具，

而是一種讓 Agent 從「只會回答」變成「可以實際進入網頁工作」的基礎能力。

它代表 Agent 開始擁有：

- 網頁視野
- 點擊能力
- 表單互動
- 網頁任務執行
- 跨網站資料收集
- 對真實網路環境的操作權限

對 Cloudhaven 來說，這接近鉗鉗未來的「網路手腳」。

如果沒有這一層，Agent 很容易停留在聊天介面。

有了這一層，Agent 才可能真的去查旅行社、比飯店、觀察遊戲頁面、讀文件、整理資料。

────────────────
## 🤔 為什麼重要？

Browser Agent Infrastructure 很重要，因為它讓 Agent 的工作邊界從文字回答擴張到真實網頁操作。

這代表：

- Agent 可以開始處理真實任務，而不是只提供建議
- 使用者可以把「幫我查」變成「幫我進去看、比較、記錄」
- Cloudhaven 的 Skill 可以開始呼叫瀏覽器能力完成外部任務
- 權限、安全、白名單、觀察與回報會變得更重要

這是 Agent 從腦袋長出手腳的關鍵物種。

────────────────
## 🛠️ 代表工具
- [BrowserAct](../ai-tools/browseract.md) — 給 AI agent 操作真實網頁的瀏覽器層工具候選。

────────────────
## 🧬 演化狀態
🌱 Seed / 萌芽

目前屬於可玩但要小心的階段。

原因：

- 能力很關鍵
- 但瀏覽器操作牽涉權限與安全
- 真實網頁容易變動
- Agent 需要清楚的白名單、回報與人類確認機制

可選狀態：

- Seed 🌱：剛出現，只有少數早期案例
- Emerging 🌿：開始有多個案例，方向變清楚
- Growing 🌳：生態快速長大，開始影響主流使用方式
- Mature 🌲：已經成熟，成為穩定基礎能力

────────────────
## 📈 觀察重點
- 是否能穩定操作真實網頁，而不是只跑 demo
- 是否支援可觀察、可回放、可限制權限的安全設計
- 是否適合接進鉗鉗的瀏覽器工作流程
- 是否能支援旅行社、遊戲商店、GitHub、文件頁等真實任務
- 是否能和 Decision / Skill / Judge 流程配合
