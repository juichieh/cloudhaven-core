# 🦁 Data Agent
> 💬 把網頁、API 與散亂資料整理成 AI 能吃的乾淨上下文。

────────────────
## 🏷️ 能力類型
📦 Data Agent

AI 資料工具／Web API／資料前處理／Context Layer。

────────────────
## ✨ 一句介紹
Data Agent 是負責抓取、補強、清洗、解析與整理資料，讓 AI 更容易理解網路內容與外部資訊的 Agent 物種。

────────────────
## 🧠 這是什麼能力？

Data Agent 不是單純操作瀏覽器，也不是長期記憶。

它比較像 AI 工作流裡的「資料消化系統」。

它負責把外部資訊變成 AI 可以使用的上下文，例如：

- 網頁內容擷取
- API 資料整理
- Metadata 補強
- 網頁正文抽取
- 商品、活動、旅遊資料解析
- 把非結構化內容轉成可比較、可搜尋、可餵給模型的資料

如果 Browser Agent 是手腳，Memory Agent 是接力記憶，

那 Data Agent 就是把外部資料處理成 AI 能吃的胃。

────────────────
## 🤔 為什麼重要？

AI Agent 要做真實任務時，常常不是缺模型，

而是缺乾淨、可用、可比較的資料。

Data Agent 的價值在於：

- 幫 Agent 工作流做前處理
- 降低網頁雜訊
- 把資料整理成適合 Decision / Skill / Judge 使用的格式
- 支援旅遊、活動、品牌研究、商品比較等任務
- 讓 AI 不只「看見資料」，而是能真正理解與使用資料

這類能力會是 Cloudhaven 未來 Travel、Research、Brand、Game 等 Room 的重要基礎零件。

────────────────
## 🧬 演化狀態
🌱 Seed / 萌芽

目前屬於值得觀察的早期工具方向。

原因：

- 工具類更新很快
- Web data / context API 仍在快速迭代
- 真正價值要看能否穩定餵給 Agent 工作流
- 適合先收藏觀察，不急著定為核心基礎設施

可選狀態：

- Seed 🌱：剛出現，只有少數早期案例
- Emerging 🌿：開始有多個案例，方向變清楚
- Growing 🌳：生態快速長大，開始影響主流使用方式
- Mature 🌲：已經成熟，成為穩定基礎能力

────────────────
## 📈 觀察重點
- 是否能穩定抽取網頁正文與關鍵資訊
- 是否能補強 metadata、價格、地點、時間、商品等資料
- 是否適合旅遊、活動、品牌研究、商品比較任務
- 是否能和 Browser Agent / Memory Agent 串接
- 是否能輸出適合 Decision / Judge 使用的乾淨資料
- 是否比一般爬取、搜尋或 reader 工具更適合 Agent 工作流
