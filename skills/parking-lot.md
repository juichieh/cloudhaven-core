# Parking Lot Skill

## Purpose

`PARKING_LOT.md` prevents useful ideas from interrupting active work.

It answers:

> What should be remembered for later without changing the current plan?

## Core Rule

Every active project may keep one root-level file:

```text
PARKING_LOT.md
```

Use it for ideas, experiments, possible revisions, unresolved questions, and paused directions that are worth remembering but are not approved work.

## Use It When

- a new idea appears during construction
- a side experiment would interrupt the current phase
- something may need revision later
- an interesting direction is not mature enough for a Proposal
- the user or Agent wants to preserve a thought before switching tasks

## Do Not Use It For

- current project state: use `CHECKPOINT.md`
- approved work: use the project workflow or Project Office
- permanent decisions: use a Decision Log
- detailed research: use `research/`
- confirmed future work with priority: use the project Backlog or Roadmap

## Writing Rules

- Keep entries short.
- State why the item matters.
- Add context or a related path only when needed.
- Do not silently promote an item into construction.
- Remove or move items when they become approved, rejected, completed, or obsolete.
- Do not split Parking Lot into more systems until real usage becomes difficult.

## Relationship to Checkpoint

```text
CHECKPOINT  = the active mainline and next action
PARKING LOT = useful thoughts that must not interrupt the mainline
```

## One-line Rule

> Park the idea. Keep building the current thing.
