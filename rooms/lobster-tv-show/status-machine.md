# 🎬 龍蝦 TV Show 狀態機

龍蝦 TV Show 的狀態機採用 Cloudhaven Narrator Layer。

這代表它不需要額外設計一套冷冰冰的狀態機文案，而是把真實工作狀態轉成節目式、角色化、可理解的狀態解說。

────────────────

## 核心判斷

> 龍蝦 TV Show 的狀態機 = Narrator Layer 的節目化輸出。

也就是：

```text
Agent Engine 真實狀態
↓
Narrator Layer
↓
龍蝦 TV Show 狀態字幕 / 口播 / 等待畫面
```

────────────────

## 為什麼這樣做

一般狀態：

```text
正在搜尋中...
正在整理中...
正在分析中...
```

雖然清楚，但沒有節目感。

龍蝦 TV Show 需要的是：

```text
🦞「今天怪東西有點多，我先把真的離譜的挑出來。」
```

或：

```text
🦞「我先翻官方資料，再去偷聽大家怎麼吵。」
```

這樣使用者仍然知道目前正在做什麼，但等待過程會更像一集節目，而不是單純看進度條。

────────────────

## 狀態來源

Narrator 不應該亂編。

它只能根據 Engine 提供的高階事件生成內容，例如：

```json
{
  "activity": "research_episode",
  "target": "AI 怪東西早報",
  "progress": 45,
  "found_items": 12
}
```

可以生成：

> 🦞「目前找到 12 個候選怪東西，我先把普通怪跟真的怪分開。」

不能生成：

> 🦞「我正在看 Reddit。」

除非 Engine 真的回報 Reddit 是資料來源。

────────────────

## 顯示形式

### 1. 字幕型

```text
🦞 正在研究本集素材...
「今天怪東西有點多，我先挑真正有節目效果的。」
```

### 2. 口播型

```text
🦞「各位觀眾等一下，我先確認這不是假新聞，不然龍蝦要被做成沙拉。」
```

### 3. 專家型

```text
🦞 正在整理本集素材...
✓ 官方資料
✓ 社群討論
✓ 產品頁
目前正在交叉驗證...
```

────────────────

## 與 Narrator Layer 的關係

詳細架構請見：

```text
architecture/narrator-layer.md
```

龍蝦 TV Show 不重新發明狀態機。

它只是把 Narrator Layer 套上節目語氣與龍蝦人格。

────────────────

## 一句話

> 龍蝦 TV Show 的狀態機不是 Loading，而是 Narrator Layer 的節目口播模式。
